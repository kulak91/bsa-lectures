2. Open up [**GitHub**](https://github.com/) and sign in. Create a new repository named **"homepage"** (e.g. **your-username/homepage**), set description to "My very own personal website. Basically this is just a r√©sum√©.", initialize with **README**, ignore **.gitignore**, but choose **"MIT License"**

1. Check current status with `git status`

2. Create a new branch and checkout to it with `git checkout -b patch/README.md`.

3. Edit README.md ‚Äî add "**üßôHocus-pocus!**" ‚Äî and stage it with `git add README.md`

4. Commit staged file with `git commit -m "Abracadabra"`

5. Upload your local changes with `git push`. If it fails, it's most likely because there's no
   such branch on the remote server. Follow Git suggestion and use `--set-upstream origin patch/README.md` or
   set the remote branch explicitly `git push origin patch/README.md`.

6. Look at the current history with `git log`.

7. Remove last line from the README.md and see the difference!
   Stage README.md file with `git add README.md` and commit with `git commit -m "Undobracadabra"`.

8. Revert the latest commit with `git revert HEAD`.

9. Look at the current history with `git log` again.

10. Upload your local changes with `git push`.

11. Switch to the `main` branch back - `git checkout main`.

   <div className="bubble small subtle">

2. Add the following tasks to the **"Resume"** project and **convert them to issues** ‚Äî  use GitHub's **Labels**, **Assignees**, and **Milestones** features at this point _([**Mastering Issues**](https://guides.github.com/features/issues/) explains why and how)_:
    Add given **descriptions** to the tasks. (Bold instructions are important - don't ignore them üòâ)

   <div className="bubble small subtle">
     Making clear description of a task is a good skill to have. It'll help you
     to set the requirements better and get along with a project manager.
     Besides, a lot of useful discussions are taking place in comments to GitHub
     issues, so it's helpful to know how to work with them. (But StackOverflow

   1. **Create an intro**
      Add an intro about yourself: who are you, what is your strength, why are you here.
   2. **Create a contact info block**
      Add links to social networks (LinkedIn, Facebook, Instagram), phone number, email, Telegram username, etc.
   3. **Create a timeline**
      Add a list of all your activities, achievements, work experience. Add as much info as you can remember. In case you don't have working experience tell about your pet-projects or student projects. Don't be shy :)
   4. **Create additional info blocks**
      You can make your resume less official by adding more entertaining info, like your hobbies and interest, or what do you like, some interactive content or joke, easter egg, etc.

3. Move tasks to <strong>"In progress"</strong> while you're working on them. Leave comments under issues, assign proper labels and milestones (provide at least 1 item of each type for every required issue), close issues and tasks when done.
