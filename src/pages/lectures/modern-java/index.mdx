### Стиль написання коду

Стиль написання коду – це набір принципів для конкретної мови програмування, які рекомендують стиль та методи програмування для кожного аспекту програми, написаної на цій мова. Ці угоди зазвичай охоплюють організацію файлів, відступи, коментарі, оголошення, оператори, пробіли, згоди стосовно імен, методи програмування, принципи програмування, практичні правила програмування, передові методи архітектури і т.д. Це керівні вказівки стосовно якості структури програмного забезпечення. Програмістах наполегливо рекомендується дотримуватися цих рекомендацій, для того, щоб поліпшити читабельність вихідного коду та полегшити підтримку програмного забезпечення.

В Java немає єдиного стилю написання коду, замість цього – кожен проект обирає стиль під власні потреби. Однак, існує три популярних стиля:

- Google style guide
- Oracle code conventions
- Spring code style

### Фреймворки та бібліотеки

`Фре́ймворк — програмна платформа, що визначаюча структура програмної системи; програмне забезпечення, що полегшує розробку та об’єднання різних компонентів великого програмного проекту.` - Wikipedia

<img src="https://pbs.twimg.com/media/DvIFxwfXcAAEyWL.jpg" />

Фреймворки надають необхідну інфраструктуру для проекту, тим самим зменшуючи кількість бойлерплейту та збільшуючи швидкість розробки.
Від бібліотеки фреймворк відрізняється використанням принципу інверсії управління - прикладний код, написаний програмістами, викликається із коду фреймворку, що дозволяє абстрагуватися від багатьох низькорівневих операцій.


Приклади Java фреймворків і бібліотек:

### Статичні аналізатори (aka лінтери)
Статичний аналізатор – інструмент, який слідкує за коректністю програми, не запускаючи її. Інструменти для статичного аналізу можуть знаходити розповсюджені помилки та антипатерни, які компілятор пропустив. Також, вони можуть слідкувати за відповідністю стилю написання коду, тим самим спрощуючи  життя програмістам.

В **Java** для статичного аналізу використовуються такі інструменти:

- PMD
- Cobertura
- FindBugs
- Jacoco

### Package management

Пакетні менеджери дозволяють перевикористовувати існуючий код та бібліотеки з легкістю. "Зовнішній" по відношенню до проекту код називається `залежністю`.
В світі Java активно використовують два пакетних менеджера: **Gradle** і **Maven**

<br />


| Gradle                                       | Maven                                                  |
| -------------------------------------------- |:------------------------------------------------------:|
| Gradle Build time is short and fast.         | Maven performance is slow as compared to Gradle.       |
| Gradle Scripts are shorter and cleaner.      | Maven scripts are a bit lengthy as compared to Gradle. |
| DSL based                                    | XML-based                                              |
| Supports incremental compilation             | Does not supports incremental compilation              |

<br />

#### Приклади gradle команд

| Команда                                     | Опис                                             |
| -------------------------------------------- |:------------------------------------------------------:|
| gradle build                                 | збірка проекту, запуск `assemble` и `check` задач      |
| gradle clean build                           | збірка проекту з нуля, без використання кешу          |
| gradle test                                  | запуск тестів                                          |
